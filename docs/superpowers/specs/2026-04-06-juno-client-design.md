# Juno React Native Client — Design Spec

**Date:** 2026-04-06
**Status:** Approved

## Overview

Juno is an AI stylist mobile app. Users photograph clothing items and receive structured editorial reviews from AI stylist voices. This spec covers the React Native client that connects to the existing Supabase backend (edge functions, auth, storage, database).

## Tech Stack

- **React Native + Expo** (managed workflow, SDK 52+)
- **Expo Router** — file-based navigation
- **React Query (TanStack)** — server state (reviews, profile, unlocks, wardrobe)
- **Zustand** — local UI state (onboarding progress, gallery selection)
- **Supabase JS** — auth, database, storage
- **RevenueCat SDK** — subscription management

## Visual Direction

Minimal/Clean with a luxury brand feel (Apple, Aesop). Dark mode only. Editorial typography. No cards-in-cards, no shadows, no gradients. Content-forward, generous whitespace.

## Screen Structure

```
(tabs)
  ├── Home (bento grid of review history, empty state)
  ├── Scan (gallery-first picker, camera secondary)
  └── Settings (profile, voice selection, account)

(stack)
  ├── Review Card (full scrollable review)
  ├── Onboarding (2-3 intro screens + optional profile)
  └── Style Quiz (visual picker, unlocks at 3 reviews)
```

## Screen-by-Screen Design

### Onboarding (first launch only)

- 2-3 swipeable full-screen pages, minimal text, one concept each:
  - "Photograph any item"
  - "Get a stylist's honest take"
  - "Build your style over time"
- Final screen: optional quick profile — height, build, gender preference as tappable pill selectors (not text fields)
- Skip button always visible, "Get Started" at the end
- After completion → Home screen

### Home Screen

**Empty state:**
- Centered illustration or subtle animation
- "Scan your first item" prominent button
- Clean, inviting, not lonely

**With history (bento grid):**
- Latest review: large tile with item photo + score overlay
- Older reviews: smaller tiles in mixed-size grid layout
- Visual hierarchy — recent is prominent, older recedes
- Tap any tile → opens that review card
- New scan accessible via bottom tab

### Scan Screen

**Gallery-first approach:**
- Opens photo library picker immediately
- Small camera icon in corner to take a new photo instead
- After photo selection → loading state → review card appears
- No live camera preview by default

### Review Card

**Full scrollable editorial layout:**
- Hero photo of scanned item at top
- Large Pitchfork-style score number (e.g., "8.5") with subtle color tint:
  - Green `#7A9A6D` for 7.0+
  - Neutral `#8A8781` for 5.0–6.9
  - Amber `#B8A45C` for below 5.0
- Item name in Instrument Serif, brand + category below
- Sections flow like a magazine column with thin hairline dividers
- Section headers: tiny uppercase, wide letter-spacing, muted color
- Sections: First Impression, Fit For You, Style Read, Originality, Construction (with inline price), Color Story, Occasion Fit, Pairing, Alternatives, Critique, Final Word
- **Final Word** in italic serif — editorial sign-off
- **Pairing section:** horizontally scrollable outfit cards with color-blocked item swatches (tee/pants/shoes shapes with + signs between them), title + vibe description per card
- **Alternatives section:** split layout with brand comparisons
- AI disclosure at bottom ("Generated by AI — opinions are Juno's, not yours.")
- Voice tag showing which stylist voice was used

**Bottom actions:**
- Back arrow (top left)
- "Scan Again" button (bottom)
- "Save to Wardrobe" action (bottom) — ties into wardrobe progressive unlock at 8 reviews

### Settings Screen

- **Profile:** height, build, gender preference (editable pill selectors)
- **Voice selection:** shows all 3 voices (Parisian Editor, Sassy Best Friend, Runway Critic). Locked until 5 reviews in one voice, shows progress toward unlock.
- **Account:** export data, delete account
- No notification settings (silent approach — no push notifications)

### Style Quiz (unlocks at 3 reviews)

- Full-screen visual picker
- Grid of outfit photos
- "Pick 5 outfits you'd wear" flow
- Tap to select, visual feedback on selection
- Results feed into future review personalization (profile data)

## Progressive Unlocks

| Feature | Unlock Threshold | Mechanism |
|---------|-----------------|-----------|
| Style Quiz | 3 reviews | unlock_events table check |
| Voice Swap | 5 reviews in one voice | unlock_events table check |
| Wardrobe | 8 reviews | unlock_events table check |

Unlocks are checked via the existing `unlock-check` edge function. The client calls this after each review and shows a celebration moment when a new feature unlocks.

## Data Flow

### Auth
- Supabase Auth with Apple + Google sign-in
- Anonymous sign-in enabled for trying the app before committing
- Auth state managed by Supabase, exposed via React context provider

### Review Pipeline (client perspective)
1. User picks image from gallery
2. Client uploads image to Supabase Storage (`scan-uploads` bucket)
3. Client calls `review` edge function with image path
4. Edge function returns structured review JSON
5. Client caches review via React Query
6. Client calls `unlock-check` to see if new features unlocked

### Server State (React Query)
- Reviews list (home screen bento grid)
- Single review (review card)
- User profile
- Unlock status
- Wardrobe items (saved reviews)
- Subscription status (via RevenueCat)

### Local State (Zustand)
- Onboarding completion flag
- Gallery/camera selection state
- Style quiz progress (before submission)

## Visual Design System

### Colors (dark mode only)
| Token | Value | Usage |
|-------|-------|-------|
| background | `#0D0D0D` | App background |
| surface | `#111111` | Screen backgrounds |
| surfaceElevated | `#161616` | Cards, tiles |
| textPrimary | `#E8E6E1` | Headlines, primary content |
| textSecondary | `#C8C5BF` | Body text |
| textMuted | `#555555` | Labels, placeholders |
| divider | `#1E1E1E` | Hairline separators |
| scoreGreen | `#7A9A6D` | Score 7.0+ |
| scoreNeutral | `#8A8781` | Score 5.0–6.9 |
| scoreAmber | `#B8A45C` | Score below 5.0 |

### Typography
| Style | Font | Size | Weight | Usage |
|-------|------|------|--------|-------|
| headline | Instrument Serif | 24px | 400 | Item names, section titles |
| score | Instrument Serif | 72px | 400 | Review score number |
| finalWord | Instrument Serif Italic | 17px | 400 | Final Word section |
| body | Inter | 14px | 300 | Review body text |
| sectionHeader | Inter | 9px | 500 | Section labels (uppercase, 0.16em tracking) |
| meta | Inter | 12px | 300 | Brand, category, price |
| caption | Inter | 10px | 400 | AI disclosure, voice tag |

### Spacing
- Horizontal padding: 24px
- Section padding: 24px vertical
- Dividers: 1px hairline, `#1E1E1E`
- No card borders, no shadows, no gradients

### Interactions
- Subtle haptic feedback on save/scan actions
- Smooth scroll on review card (no snap points)
- Bento grid tiles: slight scale animation on press
- Page transitions: horizontal slide (native feel)

## Notifications

Silent. No push notifications. No reminders. The user comes back when they want to. This aligns with the luxury brand positioning — respectful of the user's attention.

## Subscription (RevenueCat)

- Free tier: limited reviews per week (configured via app_config)
- Paid tier: higher daily cap, access to Sonnet model instead of Haiku
- RevenueCat SDK handles purchase flow, receipt validation
- Backend webhook (`webhook-revenuecat`) syncs subscription status to Supabase

## GDPR Compliance

- Export account: calls `export-account` edge function, returns all user data as JSON
- Delete account: calls `delete-account` edge function with `{"confirm": true}`, removes all data including storage objects

## Out of Scope (v1)

- Light mode
- Social features (sharing reviews between users)
- Push notifications
- Offline support
- Tablet/iPad layout optimization
- Web version
